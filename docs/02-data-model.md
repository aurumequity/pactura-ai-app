## Firestore Schema (v1 Frozen)

All tenant data is nested under:

organizations/{orgId}

Naming convention:
- camelCase for all fields
- Timestamp fields end with At
- Status fields use lowercase enums

---

## ID Strategy

Pactura uses the same UUID values across Firestore and Cloud SQL.

- orgId is a UUID generated by the API at org creation time
- documentId is a UUID generated by the API at document creation time
- Firestore document paths use orgId and documentId
- Cloud SQL stores org_id and documents.id as UUID columns

## Cloud SQL Schema (Postgres + pgvector)

Cloud SQL stores chunk text and vector embeddings used for retrieval.

Tenant isolation rule  
Every table includes `org_id`. Every query must filter by `org_id`.

---

### Extensions

```sql
CREATE EXTENSION IF NOT EXISTS vector;
````

---

## documents

Purpose
Canonical document record for retention and cascading deletes.

```sql
CREATE TABLE documents (
  id UUID PRIMARY KEY,
  org_id UUID NOT NULL,
  title TEXT NOT NULL,
  storage_path TEXT NOT NULL,
  mime_type TEXT,
  size_bytes BIGINT,
  uploaded_by_user_id UUID NOT NULL,
  status TEXT NOT NULL DEFAULT 'ready',
  retention_expires_at TIMESTAMPTZ,
  deleted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_documents_org
  ON documents(org_id);

CREATE INDEX idx_documents_org_status
  ON documents(org_id, status);

CREATE INDEX idx_documents_org_retention
  ON documents(org_id, retention_expires_at);
```

Notes

* `deleted_at` supports soft delete tracking.
* Hard deletes cascade to chunks and embeddings.

---

## doc_chunks

Purpose
Stores chunked document text and page references.

```sql
CREATE TABLE doc_chunks (
  id UUID PRIMARY KEY,
  org_id UUID NOT NULL,
  document_id UUID NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
  chunk_index INT NOT NULL,
  page_start INT,
  page_end INT,
  content TEXT NOT NULL,
  content_sha256 TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_chunks_org_doc
  ON doc_chunks(org_id, document_id);

CREATE UNIQUE INDEX ux_chunks_doc_index
  ON doc_chunks(document_id, chunk_index);
```

Notes

* `content` is the chunk text.
* `page_start` and `page_end` support citation rendering.

---

## doc_embeddings

Purpose
Stores vector embeddings for similarity search.

Adjust vector dimension to match your embedding model.
Below assumes dimension 1536.

```sql
CREATE TABLE doc_embeddings (
  id UUID PRIMARY KEY,
  org_id UUID NOT NULL,
  document_id UUID NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
  chunk_id UUID NOT NULL REFERENCES doc_chunks(id) ON DELETE CASCADE,
  embedding vector(1536) NOT NULL,
  model TEXT NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_embeddings_org_doc
  ON doc_embeddings(org_id, document_id);
```

Vector index using ivfflat for approximate nearest neighbor search:

```sql
CREATE INDEX idx_embeddings_vec_ivfflat
ON doc_embeddings
USING ivfflat (embedding vector_cosine_ops)
WITH (lists = 100);
```

Operational notes

* Run `ANALYZE doc_embeddings;` after bulk loads.
* Tune `lists` based on dataset size.
* All similarity queries must filter by `org_id`.

## Tenant-Safe Retrieval Queries

### Document-scoped top-K retrieval

```sql
SELECT
  e.chunk_id,
  (1 - (e.embedding <=> $1)) AS score
FROM doc_embeddings e
WHERE e.org_id = $2
  AND e.document_id = $3
ORDER BY e.embedding <=> $1
LIMIT $4;
```

### Chunk fetch by IDs

```sql
SELECT
  c.id,
  c.document_id,
  c.chunk_index,
  c.page_start,
  c.page_end,
  c.content
FROM doc_chunks c
WHERE c.org_id = $1
  AND c.id = ANY($2::uuid[]);
```

Rules

* Never query embeddings without `org_id`.
* Never fetch chunks without `org_id`.

## Retention and Hard Delete

Retention sweep candidate query:

```sql
SELECT id, org_id, storage_path
FROM documents
WHERE deleted_at IS NULL
  AND retention_expires_at IS NOT NULL
  AND retention_expires_at < now()
LIMIT 100;
```

Hard delete:

```sql
DELETE FROM documents
WHERE id = $1
  AND org_id = $2;
```

Cascade deletes automatically remove chunks and embeddings.

## Naming Convention

Firestore uses camelCase.
SQL uses snake_case.

Mapping examples:

* Firestore `orgId` → SQL `org_id`
* Firestore `documentId` → SQL `documents.id`
* Firestore `storagePath` → SQL `storage_path`
